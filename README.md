# 🛠 Frontend Development Guide | Hackathon

## 🚀 Быстрый старт (Локальная разработка)

**1. Клонируйте репозиторий и установите зависимости:**
```bash
npm install
```

**2. Настройте переменные окружения:**
Создайте файл `.env` в корне проекта (рядом с `package.json`).
```bash
cp .env.example .env
```
> **Важно:** Убедитесь, что в `.env` стоит `VITE_USE_MOCKS=true`, пока бэкенд не будет готов к интеграции.

**3. Запустите сервер разработки:**
```bash
npm run dev
```
Приложение будет доступно по адресу: `http://localhost:3000`

## 📂 Структура проекта (Архитектура)

Проект построен на **Vue 3 (Composition API) + Vite**. Вся логика строго разделена по директориям, чтобы избежать мёрдж-конфликтов.

```text
frontend/
├── public/               # Статика, которая не обрабатывается Vite
├── src/                  # Исходный код
│   ├── api/              # 🌐 Сетевой слой (Axios client, utils, mocks)
│   ├── assets/           # 🖼 Медиафайлы (картинки, SVG, шрифты)
│   ├── components/       # 🧩 Переиспользуемые UI-компоненты
│   ├── composables/      # 🪝 Кастомные хуки Vue (вынесенная логика)
│   ├── router/           # 🗺 Маршрутизация и защита роутов
│   ├── stores/           # 📦 Стейт-менеджмент (Pinia)
│   ├── types/            # 🟦 Глобальные TS интерфейсы
│   ├── views/            # 📄 Компоненты-страницы (Home, Login)
│   ├── App.vue           # Корневой компонент
│   ├── main.ts           # Точка входа (Vue, Pinia, Router)
│   ├── style.css         # Глобальные стили
│   └── vite-env.d.ts     # Декларации TypeScript
├── .env                  # Локальные конфиги (ИСКЛЮЧЕН ИЗ GIT)
├── .env.example          # Шаблон переменных окружения
└── vite.config.ts        # Конфиг сборщика
```

## ✅ Что уже реализовано

### 1. Авторизация и Стейт (`useAuthStore.ts`)
* Реализовано безопасное хранение JWT-токена.
* Реактивное свойство `isAuthenticated` для моментального обновления UI.
* Метод `logout()`, очищающий сессию и перенаправляющий пользователя.

### 2. Защита маршрутов (`router/index.ts`)
* Настроен глобальный Navigation Guard (`router.beforeEach`).
* Страницы с `meta: { requiresAuth: true }` недоступны без токена (автоматический редирект на `/login`).
* Авторизованные пользователи не могут попасть на страницу входа.

### 3. Умный HTTP-клиент (`api/client.ts`)
* **Авто-подстановка токена:** Axios автоматически берет токен из Pinia и добавляет заголовок `Authorization: Bearer` ко всем запросам.
* **Глобальный перехват 401:** Если токен истек, клиент автоматически вызывает `logout()` и отправляет на страницу входа.

### 4. Система Моков (Заглушек)
Вы можете разрабатывать фичи без бэкенда!
* Если в `.env` указано `VITE_USE_MOCKS=true`, API-функции возвращают захардкоженные данные с искусственной задержкой сети.
* Когда бэкенд готов, просто поменяйте флаг на `false` — код компонентов останется нетронутым.
